## Known Issues

### 1. While Loop Infinite Loop (Critical)

**Symptom:** VM enters infinite loop when executing while loops.

**Example:**
```kai
x := 5
while x > 0 {
  x := x - 1
}
```

**Expected:** Loop executes 5 times, exits
**Actual:** Loop never exits

**Suspected Causes:**
1. Jump offset calculation error in `compileWhileLoop()`
2. Condition not being re-evaluated each iteration
3. Variable mutation not updating VM state properly

**Debugging Steps Needed:**
1. Add disassembly output to see generated bytecode
2. Trace VM execution step-by-step
3. Verify register values at each iteration
4. Check if `JUMP_IF_FALSE` is working correctly

### 2. Missing Array Indexing

**Problem:** Cannot access array elements by index.

**Needed:** `GET_ELEM arr, idx, dest` instruction

### 3. No Call Stack

**Problem:** Function calls cannot be implemented without call stack.

**Needed:**
- Stack frame management
- Parameter passing
- Return address handling

## Architecture Decisions

### Register-Based vs Stack-Based

**Choice:** Register-based VM with 32 virtual registers

**Rationale:**
- Easier to generate efficient code
- Closer to real hardware
- Better performance potential
- Simpler register allocation than stack machines

### Bytecode Format

**Instruction Format:** `Opcode operand1, operand2, ...`

**Example:**
```
ADD r0, r1, r2    # r0 = r1 + r2
JUMP_IF_FALSE r3, 5  # if !r3 then jump forward 5 instructions
```

### Compilation Strategy

**Approach:** Direct AST to bytecode (no intermediate IR yet)

**Rationale:**
- Simpler for Phase 5.1
- Can add SSA IR later (Phase 5.3+)
- Keeps codebase manageable
